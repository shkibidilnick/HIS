## Что делает программа

В каждой части:
1. Формирует таблицу истинности / таблицу переходов.
2. Записывает функции в совершенной форме (СДНФ или СКНФ согласно варианту).
3. Строит карты Карно для каждой выходной функции.
4. Минимизирует функции табличным методом (с учётом don't care, где они есть).
5. Выводит финальные формулы и текстовое описание схемы для Logisim.
6. Проводит верификацию полученных формул эмуляцией.

## Использованные технологии

- Чистый Python 3 без сторонних библиотек.
- Из стандартной библиотеки используются только `itertools.product` и
  `itertools.combinations` — это базовые комбинаторные генераторы, а не
  готовое решение задачи минимизации.

## Соответствие принципам качественного кода

- **KISS** — алгоритм минимизации реализован прямым перебором простых импликант
  без оптимизаций; для функций до 4 переменных это работает мгновенно.
- **DRY** — единый модуль `karnaugh_map.py` обрабатывает любые функции от
  3–4 переменных, минимизация ДНФ и КНФ через общий механизм с параметром
  `target_value`.
- **SRP** — `KarnaughMap` отвечает только за хранение и отображение карты,
  `Minimizer` — только за минимизацию, `*Synthesizer` — только за оркестрацию.
- **YAGNI** — поддерживается ровно вариант задания, без избыточной
  конфигурируемости.

## Демонстрация в Logisim

### Часть 1 (8-разрядный сумматор)
- Подсхема `ODS3` (1 разряд) собирается по полученным формулам S и P.
- 8 копий в каскад через перенос. Демонстрация: 8 + 6 = 14
  (A=00001000, B=00000110, Sum=00001110, Cout=0).

### Часть 2 (преобразователь BCD+5)
- Переносится без масштабирования. Проверка по таблице:
  вход 0 → выход 5, вход 9 → выход 14.

### Часть 3 (счётчик на 8 состояний)
- 3 Т-триггера + комбинационная схема возбуждения.
- При запуске тактирования последовательно выдаёт 000, 001, ..., 111, 000.
